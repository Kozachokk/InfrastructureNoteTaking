GIT

Version control software(VCS) is software that is used to automatically control and track changes to project(repositories), in order to efficiently store multiple versions of them
Git is an open-source distributed version control system(DVCS) meaning that it's a VCS, but every person that downloads a copy of the repository, downloads the whole history on their machine
Commonly used commands:

  • git init: creates a local git repository
  • git add: adds files to the staging area
  • git commit -m "message": saves the changes to the local repository
  • git status: show the state of the working directory and staging area

Branches are also an important concept in version control since they are essentially a copy of the project on which you can work without changing anything in the main branch that has the working code,
so you can test and play with new things without breaking anything
Commonly used commands:

  • git branch <name>: creates a new branch
  • git switch <branch_name>: Switches to another branch
  • git merge <branch_name>: Merges current branch with another one
  • git checkout <name>: Swithces between differet version of the project
  • git branch: List all branches
  • git branch -v: List all branches in verbose mode
  • git branch -d <branch_name>: Deletes the branch
  • git branch -m <Old branch name> <New branch name>: Renames the branch

There are also remote repositories which are hosted online so anyone can access them, they act as a central hub, popular hosts include: Github, Gitlab, Bitbucker
Commonly used commands:

  • git clone <url>: Creates a copy in a local repository
  • git push origin <branch_name>: Uploads the local changes to the origin(remote repository)
  • git pull origin <branch_name>: Downloads the branch from the remote repository to the locac one
  • git fetch origin: Downloads objects and refs from another repository
